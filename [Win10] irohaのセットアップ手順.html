環境

  • Windows10 Home バージョン 2004 [build 19041] 以降
  • WSL2が導入済であること
  • Docker for Windows version 19.03.12 以降が導入済であること
  • git が導入済であること

セットアップ手順

2020.09.23 現在のバージョンでインストールする

以下のバージョンがインストールされます

service version
Iroha Board 0.10.4
CakePHP 2.10.20
PHP 9.4.3
MySQL 8.0.21

環境構築

  1. プロジェクトフォルダ作成

    • iroha boardプロジェクト用のフォルダを作成する
      • 例：
      mkdir /projects
mkdir /projects/iroha

  2. iroha boardのダウンロード

    • プロジェクトフォルダに、横山によるソースセットを丸ごとコピーする
      （2020.09.23 現在は https://github.com/nominalrune/iroha-board-Dockerized.git に公開している。社内向けに、安定的な場所に移動予定）
    • 例: GitHubからリポジトリをクローンして、ブランチを作成
      git clone https://github.com/nominalrune/iroha-board-Dockerized.git /projects/iroha
cd /projects/iroha
git checkout -b lbProjectTest

  3. 設定

    1. MySQLのパスワード設定
    • ルートパスワードを設定する（デフォルトではroot）
      1. MySQL側の設定
        1. docker-compose.yml内の環境変数MYSQL_ROOT_PASSWORDの値を変更
          • 例:
            code docker-compose.yml
# ~~中略~~
     mysql: #MySQLの項目を見つける
	image: mysql:8.0.21
	restart: unless-stopped
	networks:
	    - iroha_net
	environment:
	    - MYSQL_DATABASE=irohaboard
	    - MYSQL_ALLOW_EMPTY_PASSWORD=yes
	    - MYSQL_ROOT_PASSWORD=root # パスワードの変更はここ
	    - TZ=Asia/Tokyo
	volumes:
	    - ./apache/sql:/var/lib/mysql:rw
	tty: true
	stdin_open: true
# ~~中略~~
      2. iroha board側の設定
        1. プロジェクトディレクトリ以下./apache/html/app/Config内にある、database.phpの'password'変数をMySQLでのパスワードに合わせる
          • 例:
            code ./apache/html/app/Config/database.php
# ~~中略~~
class DATABASE_CONFIG {
        public $default = array(
                'datasource' => 'Database/Mysql',
                'persistent' => true,
                'host' => 'mysql',
                'login' => 'root',
                'password' => 'root', # パスワードの変更はここ
                'database' => 'irohaboard',
                'prefix' => 'ib_',
                'encoding' => 'utf8'
        );
}
# ~~中略~~

イメージのビルドと起動

  1. コンテナからイメージを作成
    • irohaプロジェクトのルートディレクトリにいることを確認する。（例では/iroha）
    • 次のコマンドを打つ。
      docker-compose up --build
  2. サービスが起動するまで待つ
    • mysqlイメージが
      [System] [MY-010931] [Server] /usr/sbin/mysqld: ready for connections. Version: '8.0.21'  socket: '/var/run/mysqld/mysqld.sock'  port: 3306  MySQL Community Server - GPL.
      というメッセージを出して待機状態になれば完了

動作確認

  1. アクセス確認
    1. ウェブブラウザを開き、127.0.0.1:10080/installにアクセスする
    2. 管理者パスワード設定画面が表示されれば成功
  2. 管理者ログイン
    1. 管理者パスワード設定画面にて、好きなパスワードを入力
    2. ［インストール］ボタンを押す
    3. ［管理者ログイン画面へ］ボタンを押す
    4. ログインする
      （→［ユーザー一覧］画面が表示される）
  3. 授業コースを作成
    1. 上部ナビバーから［コース］をクリック
      （→［コース一覧］画面が表示される）
    2. ［＋追加］ボタンをクリック
      1. 「コース名」にテストコースと入力して［作成］ボタンをクリック
        （→［コース一覧］画面が表示される）
      2. ［コース一覧］画面に、作成したコース（テストコース）が表示されていることを確認する
  4. 授業コンテンツのアップロード
    1. ［コース一覧］画面にて、テストコースをクリック
      （→［コンテンツ一覧］画面が表示される）
    2. ［＋追加］ボタンをクリック
    3. 「コンテンツタイトル」にテスト映像と入力
    4. 「コンテンツ種別」は映像にする
      1. mp4動画を用意してアップロード
        • 上限2MBまでの動画ファイルがアップロードできることを確認する
    5. ［保存］ボタンを押す
    6. ［コンテンツ一覧］画面に、作成したコンテンツ（テスト映像）が表示されていることを確認する
  5. 受講者のアカウント作成
    1. ［ユーザ一覧］画面から、［＋追加］ボタンを押す
      1. 入力事項
        • 「名前」：テスト受講者
        • 「パスワード」：test
        • 「権限」：受講者
        • 「受講コース」：テスト授業
      2. ［保存］ボタンをクリック
    2. ［ユーザ一覧］画面にテスト受講者が登録されていることを確認する
  6. 受講者アカウントでのログイン
    1. 管理者アカウントからログアウト
      • ページ右上に「ログアウト」ボタンがある
    2. 「管理者ログイン」画面が表示される
    3. 「受講者ログインへ」というリンクをクリック
    4. 先ほど登録した受講者アカウントでログインする
      • ログインID：テスト受講者
      • パスワード：test
    5. 「コース一覧」にコース（テストコース）が割り当てられていることを確認する
    6. テストコースをクリック
    7. テスト授業をクリック
      • 映像が再生できることを確認する
  7. テストデータの削除
    1. 管理者アカウントでログインする
    2. 受講者データの削除
      1. ［ユーザー一覧］画面にて、テスト受講者の欄の［削除］ボタンを押す
    3. コースデータの削除
      1. ［コース一覧］画面にて、テストコースの欄の［削除］ボタンを押す
